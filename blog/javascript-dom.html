What is the DOM?

The Document Object Model (DOM) is an application that helps us navigate webpages. It serves as an interface that allows programming languages to access and manipulate objects within a web page.


When you go on your phone & you want to listen to music, do you launch your music app & manually scroll through every single song known to mankind to find that one song you feel like listening to? No, right? Or at least, I hope you wouldn't.

The take away here is: The DOM is an effective & essential tool for programmers working with webpages.


Arrays & Objects

Arrays are containers, ordered lists of information. Here are some key points about arrays:

  • Declaration: To create an array, you use the square brackets '[]' and input your elements inside.
  • Ordered Collection: Arrays store elements in a specific order.
  • Counting: When counting elements within an array, you don't begin with 1, you begin with 0.
  • Mixed Data Types: Arrays can hold different types of data in each element. You can have numbers, strings, objects, all in the same array.

Objects are also containers of information. Objects store related information in the form of 'key-value' pairs. Here are some key points about objects:

  • Declaration: To create an object, you use the curly braces '{}' and define it's properties inside.
  • Key-Value Pairs: Objects store data as pairs of keys and values.
  • Properties: Keys in objects are called properties.
  • Mixed Data Types: Like arrays, objects can hold different types of data. You can have numbers, strings and even arrays, so long as the object is formatted properly.

The take away here is: Although arrays & objects are very similar, they're not to be mixed up! They're two very different things! Use the example images above to differentiate them!


Functions

Functions are blocks of reusable code that perform a specific task. Here are some key points about functions:

  • Purpose: Functions contain instructions that get executed when you use them.
  • Parameters: Functions take in inputs, these are called parameters.
  • Return: After finishing it's job, a function can give you a return(ed) value.
  • Reusability: Once functions are defined, they can be reused multiple times through out your code.

The take away here is: Functions are handy tools that let you organize tasks & reuse code.
